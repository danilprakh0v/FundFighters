# Technical Report — FundFighters
*Курсовой проект, ФКН ВШЭ (НИС) — Прахов Данил*

## 1. Общее архитектурное решение платформы

**Client (iOS App)**
Приложение разделено на модули по экранам (Onboarding, Splash, Auth, Dashboard, Battle) по паттерну **MVVM (Model-View-ViewModel)**. 
- UI строится исключительно через код (programmatic AutoLayout), что решает проблемы мердж-конфликтов `Main.storyboard`, а также обеспечивает гранулярный пиксельный контроль над расположением элементов.
- Разработана система дизайн-токенов (`DesignSystem.swift`), концентрирующая типографику (шрифты `Golos Text`, `Inter`), отступы и корпоративные цвета проекта (например, `DS.accent` — фирменный зеленый оттенок `#1E8C62`).
- Анимации выведены на первый план (`UIKit Dynamics`, `UIView.animate` с пружинной демпинговой настройкой, сложная каскадная прозрачность во EntryAnimationViewController) для геймификации.

**Backend (ASP.NET Web API)**
Чистая архитектура (Clean Architecture / Onion) на .NET:
- **Presentation (API)**: Контроллеры для обработки HTTP-запросов. Реализован Swagger для документирования.
- **Application**: Реализация паттерна **CQRS** с использованием `MediatR`. Команды и запросы четко разделены, что упрощает масштабирование.
- **Domain**: Богатая модель предметной области (Rich Domain Model). Сущности, такие как `Enemy` и `Player`, содержат в себе бизнес-логику (например, расчет урона и XP), а не являются просто набором свойств.
- **Infrastructure**: Работа с PostgreSQL через EF Core. Настроена система миграций и репозиториев. Реализован `JwtService` для генерации токенов и `SmtpEmailService` для 2FA.

## 2. Подробности интеграций

### Авторизация и Сетевой слой
- Реализован универсальный `NetworkManager` на базе `URLSession`, поддерживающий Generic-декодирование JSON и кастомные стратегии обработки дат .NET.
- Автоматическая подстановка JWT-токена в Header (`Authorization: Bearer ...`) через `TokenManager`.
- Система поддерживает полную цепочку Auth: Регистрация -> Верификация Email -> Логин -> 2FA.

### Компонентный Dashboard
Вместо монолитного экрана, Dashboard разбит на самодостаточные "уровни":
1. `BalanceCardView` — стек общей аналитики с градиентами.
2. `SavingsGoalCardView` — витрина текущего монстра-цели. Сложный расчет констрейнтов из числа 💔 ("сердец босса").
3. `RecentActivityView` & `ExpenseChartView` & `BattleCardView` — динамические карточки с фильтрацией по дате и поддержкой реальных данных из API.

### Battle View Controller (Сайд-система сражений)
Изолированная логика `Fight` (переход на выделенный экран). Интегрирована с бэкендом: транзакции отправляются на сервер, который рассчитывает урон боссу и возвращает обновленное состояние битвы, визуализируемое через анимации тряски и изменения HP-бара.

## 3. Дальнейшие задачи и перспективы
После успешной интеграции базового функционала, приоритетными задачами являются:
1. **Push Notifications**: Уведомления о необходимости "нанести удар" (совершить сбережение) или о победе над боссом.
2. **Offline Mode**: Внедрение CoreData или Realm для кэширования данных Dashboard и возможности создания транзакций без интернета с последующей синхронизацией.
3. **Social System**: Система друзей и совместных рейдов на "мировых боссов" (общие финансовые цели).
4. **WidgetKit**: Вывод текущего прогресса цели на главный экран iOS.

Проект продемонстрировал возможности современного UIKit в создании производительных, эстетичных приложений, сопоставимых по качеству с SwiftUI, но без компромиссов на совместимость и гибкость AutoLayout.
